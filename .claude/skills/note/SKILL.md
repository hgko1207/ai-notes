# AI 노트 작성 스킬

사용자가 제공한 정보 소스(유튜브 영상, 블로그 글, 뉴스 등)를 분석하여 체계적인 마크다운 노트로 작성한다.

## 실행 절차

### 1단계: 소스 분석
- 제공된 URL이나 텍스트를 깊이 분석
- WebFetch로 웹 콘텐츠 가져오기 (URL 제공 시)
- 핵심 주장, 데이터, 인사이트 추출
- 사실 관계 및 맥락 검토

### 2단계: 분류 결정
프로젝트 폴더 구조에 맞게 가장 적합한 폴더를 결정:
- `models/` — AI 모델 관련 (특징, 비교, 활용법)
- `prompting/` — 프롬프트 엔지니어링 (기법, 템플릿)
- `tools/` — AI 도구/서비스 (리뷰, 활용 팁)
- `concepts/` — AI 핵심 개념 (기술 원리 설명)
- `trends/` — AI 업계 뉴스/트렌드 (동향)
- `tutorials/` — 실습 가이드 (따라하기)
- `resources/` — 링크/논문/강의 모음

사용자에게 분류를 확인받거나, 명확한 경우 바로 진행.

### 3단계: 마크다운 작성
아래 템플릿을 따른다:

```markdown
# 제목 (핵심을 한 줄로)

> 한 줄 요약: 핵심 내용 한 문장

## 핵심 요약
- 바쁜 사람을 위한 3~5줄 핵심 포인트

## 본문
### 소제목 1
### 소제목 2
...

## 실전 활용 / 팁 (해당 시)

## 마무리

---
**출처**: [원본 제목](URL)
**작성일**: YYYY-MM-DD
**태그**: #태그1 #태그2 #태그3
```

### 문체
- **존댓말(~입니다/합니다) 필수** — "~이다/~한다" 평서체 금지
- 블로그 독자에게 친근하고 읽기 편한 톤 유지

### 작성 원칙
1. **핵심 먼저** — 결론/요약을 글 상단에 (역피라미드)
2. **쉬운 말** — 전문 용어는 처음 등장 시 설명
3. **구조화** — 글머리 기호, 표, 번호 목록 적극 활용
4. **실용적** — "그래서 뭘 할 수 있는데?"에 답하기
5. **적절한 길이** — 핵심 집중, 장황함 제거
6. **출처 명시** — 원본 링크 항상 포함
7. **블로그 친화** — 티스토리에 바로 올릴 수 있는 수준의 가독성

### 4단계: 파일 생성
- 파일명: 소문자 영문, 하이픈 구분 (예: `claude-4-analysis.md`)
- 해당 폴더에 Write 도구로 저장
- 사용자에게 작성 완료 보고 (파일 경로 + 핵심 요약)
