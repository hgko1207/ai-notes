# AI Notes 프로젝트

## 프로젝트 목적
AI/인공지능 관련 정보를 유튜브, 블로그 등 다양한 소스에서 수집하여 체계적으로 정리하는 노트 저장소.
티스토리 블로그 발행도 고려하여, 사람들이 읽기 쉽고 정보를 빠르게 습득할 수 있는 형태로 작성한다.

## 폴더 구조 및 분류 기준
| 폴더 | 용도 | 예시 |
|------|------|------|
| `models/` | AI 모델별 특징, 비교, 활용법 | GPT-4o 분석, Claude vs Gemini 비교 |
| `prompting/` | 프롬프트 엔지니어링 기법과 템플릿 | Chain-of-Thought, 시스템 프롬프트 가이드 |
| `tools/` | AI 도구 및 서비스 리뷰, 활용 팁 | Cursor 사용법, Midjourney 팁 |
| `concepts/` | AI 핵심 개념과 기술 원리 | RAG, Fine-tuning, Transformer 설명 |
| `trends/` | AI 업계 뉴스, 트렌드, 동향 | 주간 AI 뉴스, 업계 방향성 |
| `tutorials/` | 실습 가이드, 따라하기 | API 연동 튜토리얼, 로컬 LLM 설치 |
| `resources/` | 유용한 링크, 논문, 강의 모음 | 추천 강의 목록, 논문 요약 |

## 글 작성 규칙

### 파일명
- 소문자 영문, 하이픈(`-`) 구분: `claude-4-analysis.md`
- 날짜 접두사 선택적: `2026-03-28-claude-4-analysis.md`

### 마크다운 구조
모든 글은 아래 템플릿을 따른다:

```markdown
# 제목 (핵심을 한 줄로)

> 한 줄 요약: 이 글에서 다루는 핵심 내용을 한 문장으로

## 목차 (길이가 긴 글일 경우)

## 핵심 요약
- 바쁜 사람을 위한 3~5줄 핵심 포인트
- 글 전체를 읽지 않아도 핵심을 파악할 수 있도록

## 본문
### 소제목 1
### 소제목 2
...

## 실전 활용 / 팁 (해당 시)
- 실제로 어떻게 쓸 수 있는지

## 마무리
- 핵심 정리 또는 개인 의견

---
**출처**: [원본 링크](URL)
**작성일**: YYYY-MM-DD
**태그**: 태그1, 태그2, 태그3
```

### 문체
- **존댓말(~입니다/합니다)** 사용 필수
- "~이다/~한다" 평서체 금지 — 블로그 독자와 거리감이 생김
- 티스토리 블로그 발행을 고려하여 친근하고 읽기 편한 톤 유지

### 작성 원칙
1. **핵심 먼저**: 결론/요약을 글 상단에 배치 (역피라미드 구조)
2. **쉬운 말**: 전문 용어는 처음 나올 때 간단히 설명
3. **구조화**: 글머리 기호, 표, 번호 목록을 적극 활용
4. **실용적**: "그래서 뭘 할 수 있는데?"에 답하는 내용 포함
5. **적절한 길이**: 핵심에 집중, 불필요한 장황한 설명 제거
6. **출처 명시**: 원본 소스 링크 항상 포함

### 소스 분석 프로세스
사용자가 정보(유튜브, 블로그 등)를 제공하면:
1. **분석**: 내용을 깊이 이해하고 핵심을 추출
2. **검증**: 사실 관계 확인, 누락된 맥락 보완
3. **분류**: 적절한 폴더 결정
4. **작성**: 위 템플릿과 원칙에 따라 마크다운 작성
5. **저장**: 해당 폴더에 파일 생성
