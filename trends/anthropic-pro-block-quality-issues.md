# Anthropic 위기 신호 — Claude Pro 차단 실험과 모델 품질 저하 보고서

> 한 줄 요약: Anthropic이 Claude Code를 Pro 사용자에게 차단하는 가격 정책 실험을 진행하고, 최근 모델 품질 저하 의혹에 대해 컴퓨팅 절감을 위한 변경이 있었음을 사실상 인정했습니다.

## 핵심 요약
- **Claude Code Pro 차단 실험**: 신규 Pro 가입자 약 2%에게 Claude Code 사용을 차단하는 A/B 테스트 진행, 거센 비판 직면
- **품질 저하의 원인 공식 인정**: Claude Code 추론 노력 수준을 High → Medium으로 낮춘 변경 등 3가지 변경사항이 품질에 영향
- **컴퓨팅 파워 부족 의혹**: 표면적으로는 "지연 시간 단축"이라고 설명하지만 실제로는 비용·자원 절감 압박이 작용했을 가능성
- **경쟁사 우위 시기**: GPT 5.5 출시와 Codex의 약진으로 Anthropic이 처음으로 후순위로 밀린 분위기

## 본문

### Claude Code Pro 차단 실험

#### 무슨 일이 있었나

원래 Claude Pro 구독에 포함되던 Claude Code 사용이 갑자기 차단되는 일이 일어났습니다. Pro 구독자들이 큰 비난을 쏟아냈고 Anthropic이 공식 답변을 내놨습니다.

#### Anthropic 공식 답변

Head of Growth **아몰 아발세러(Amol Avalsera)**가 직접 입장을 밝혔습니다.

> "명확성을 위해, 우리는 신규 프로슈머 가입자의 약 2%에 대해 작은 테스트를 진행했습니다."

| 항목 | 내용 |
|------|------|
| 대상 | 신규 Pro 가입자 약 2% |
| 실험 내용 | Claude Code 사용 차단 시 가입 의향 측정 |
| 목적 | 상위 요금제(Max 등) 전환율 측정 |
| 결과 후 조치 | 문구가 "Pro/Max"에서 "Max"로 변경 |

#### 의심받는 이유

A/B 테스트 자체는 일반적이지만, 다음 정황 때문에 단순 실험 이상의 의도가 있다는 의심을 받고 있습니다.

- 이미 가입 페이지 문구가 "Pro/Max"에서 "Max"로 변경됨
- Pro 사용자들은 Pro 가격에 Claude Code가 포함된 줄 알고 가입한 상태
- **상위 요금제 중심으로 재배치하려는 움직임**으로 해석됨

### Claude Code 품질 저하 — 공식 인정

최근 커뮤니티에서 "Claude 모델이 멍청해진 것 같다"는 의견이 많아지자, Anthropic이 **Claude Code 품질 보고서**를 공개했습니다.

#### 3가지 변경사항

| 일자 | 변경 내용 | 되돌린 일자 |
|------|----------|-------------|
| 3월 4일 | 추론 노력 수준 High → Medium으로 변경 (지연 시간 단축 명목) | 4월 7일 |
| 3월 26일 | 1시간 이상 미사용 세션의 이전 생각 자동 삭제 | 4월 10일 |
| ? | 시스템 메시지에 "장황함 줄이기" 지침 추가 | 4월 20일 |

#### Anthropic의 설명

표면적으로는 모두 **"지연 시간 단축"**이 목표였다는 설명입니다.

#### 시장의 추측

| 명목상 이유 | 의심되는 실제 이유 |
|------------|-------------------|
| 지연 시간 단축 | 컴퓨팅 파워 절감 |
| 응답 속도 향상 | 비용 절감 (상장 준비 영향?) |
| 사용자 경험 개선 | 서버 부하 감소 |

상장(IPO)을 준비하는 단계에서 비용 효율성을 높이려는 노력이 결국 모델 품질에 영향을 줬을 가능성이 제기됩니다.

### 왜 지금 이런 일이 벌어지나

#### 경쟁 구도의 변화

| 시기 | 상황 |
|------|------|
| 2026년 초 | Anthropic 독주, Claude Opus 4.7이 종합 1위 |
| 2026년 4월 | GPT 5.5 출시로 1위 자리 내줌, Codex 약진 |

경쟁사가 약했을 때는 Anthropic이 적극적으로 자원을 투입할 수 있었지만, 이제 OpenAI의 Codex가 강력한 대안이 되면서 비용 효율을 무시할 수 없게 됐습니다.

#### 이탈 가능성

GPT 5.5의 등장으로 Claude Code 사용자가 GPT 기반 도구로 옮겨갈 가능성이 높아지고 있습니다. Anthropic 입장에서는 사용자 유지와 비용 효율 사이의 균형을 다시 잡아야 하는 상황입니다.

## 실전 활용 / 팁

- **Pro 구독자라면**: 본인 계정의 Claude Code 사용 가능 여부를 확인하고, 차단됐다면 Anthropic에 문의하거나 Max 플랜 전환을 검토해야 합니다
- **Claude Code 품질 모니터링**: 최근 답변이 평소보다 짧거나 단순하다면, 시스템 변경 영향일 수 있습니다. 이번 보고서에 따라 4월 20일 이후 대부분 되돌려졌으므로 정상화됐는지 확인해보세요
- **대안 도구 시야 확장**: Claude Code 외에도 Codex, Cursor, GPT 5.5 직접 사용 등 여러 옵션을 두고 작업 유형별로 비교하는 것이 좋습니다
