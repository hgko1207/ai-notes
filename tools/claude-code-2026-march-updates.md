# 클로드 코드 2026년 3월 대규모 업데이트 총정리

> 한 줄 요약: Claude Code에 모바일 원격 제어(Dispatch), 메신저 연동(Channels), 클라우드 스케줄링이 추가되면서, PC를 꺼놔도 AI가 일하는 시대가 열렸습니다.

## 핵심 요약
- **Dispatch**: 휴대폰에서 데스크톱 Claude Code를 원격으로 조작
- **Channels**: 텔레그램, 디스코드 등 메신저로 Claude Code에 작업 지시
- **클라우드 스케줄링**: PC를 꺼도 클라우드에서 반복 작업 자동 수행
- **YC CEO 게리 탄의 스킬 셋업**: 제품 출시 전 과정을 스킬로 정리한 노하우 공개

## 본문

### Dispatch — 휴대폰으로 내 PC의 Claude Code 조작

Claude Code의 데스크톱 버전인 **Claude Co-Work**에 추가된 기능입니다. 핵심은 **휴대폰에서 데스크톱 컴퓨터의 Claude Code를 원격으로 조작**할 수 있다는 것입니다.

#### 작동 방식
1. 데스크톱에 Claude Code(Co-Work) 실행
2. 휴대폰에서 Claude 앱으로 Dispatch 기능 접속
3. "다운로드 폴더에서 랜딩 페이지 프로포절 PPT 열고 키포인트 가져와 줘" 같은 명령 전달
4. 데스크톱의 Claude가 파일을 열고, 정보를 추출해서 결과를 알려줍니다

#### 활용 시나리오
- 외출 중에 PC에 있는 파일 내용 확인
- 이동 중 코딩 작업 지시
- 회의 중 필요한 자료를 AI가 알아서 찾아서 전달

마치 Open Claw의 메신저 기능과 비슷하지만, Anthropic 공식 앱에서 네이티브로 지원한다는 점이 차이점입니다.

---

### Channels — 메신저로 Claude Code와 소통

Open Claw에서 먼저 인기를 얻은 기능을 Claude Code가 도입한 형태입니다. **텔레그램이나 디스코드 같은 외부 메신저**로 Claude Code에 메시지를 보내 작업을 지시할 수 있습니다.

#### 특징
- **양방향 통신**: 명령을 보내면 Claude Code가 답장도 해줍니다
- **외부 메신저 활용**: 별도 앱 없이 평소 쓰던 메신저 그대로 사용
- **연결 방법**: Claude Code 실행 중 Channels 기능으로 연결 → 메신저에서 바로 전송

#### Dispatch vs Channels 차이

| 항목 | Dispatch | Channels |
|------|----------|----------|
| 접속 방법 | Claude 공식 앱 | 텔레그램, 디스코드 등 |
| 대상 | Claude Co-Work (데스크톱) | Claude Code (터미널) |
| 느낌 | 공식 원격 제어 | 메신저 기반 자유로운 소통 |

---

### 클라우드 기반 스케줄링 — PC 꺼도 작업 수행

기존에도 로컬 PC에서 반복 작업을 스케줄링하는 기능은 있었습니다. 이번에 달라진 점은 **클라우드에서 돌아간다**는 것입니다.

#### 기존 vs 신규

| 항목 | 기존 | 신규 |
|------|------|------|
| 실행 환경 | 내 PC | **클라우드** |
| PC 꺼도 동작 | X | **O** |
| 활용 예시 | — | PR 리뷰 정기 검사, 코드 품질 모니터링 |

컴퓨터를 꺼놓고 자도 Claude가 클라우드에서 알아서 작업을 수행합니다. 예를 들어:
- Pull Request 리뷰를 정기적으로 검사
- 코드 품질 지표를 주기적으로 체크
- 특정 이벤트 발생 시 자동 알림

### YC CEO 게리 탄의 Claude Code 스킬 셋업

실리콘 밸리의 대표적 VC인 Y Combinator의 CEO **게리 탄(Garry Tan)**이 자신의 Claude Code 스킬 셋업을 공개했습니다.

#### 왜 주목할까?

게리 탄은 수많은 초기 스타트업을 검토하고 투자해온 인물입니다. 새로운 제품을 성공적으로 출시하는 노하우를 **Claude Code 스킬 형태**로 정리해 공개한 것입니다.

#### 스프린트 프로세스

제품 출시의 전 과정을 7단계로 정리했습니다:

1. **생각(Think)** — 아이디어 구체화
2. **계획(Plan)** — 실행 계획 수립
3. **구축(Build)** — 실제 개발
4. **검토(Review)** — 코드/제품 리뷰
5. **테스트(Test)** — QA 및 테스트
6. **출시(Ship)** — 배포
7. **성찰(Reflect)** — 회고 및 개선

#### 포함된 역할 스킬

| 스킬 | 역할 |
|------|------|
| Office Hours | 아이디어 검증 |
| CEO/창립자 리뷰 | 전략적 관점 피드백 |
| 엔지니어링 매니저 | 기술 아키텍처 검토 |
| 수석 디자이너 | UI/UX 리뷰 |
| 수석 엔지니어 | 코드 품질 검토 |
| 디버거 | 버그 분석 및 해결 |

이 스킬들을 사용하면 마치 **YC의 피드백을 받으며 제품을 만드는 것**과 유사한 경험을 할 수 있습니다.

> 기본 Claude Code가 못 하는 것들은 이런 스킬의 형태로 다운받아서 보강하면 훨씬 더 잘 수행할 수 있습니다.

## 실전 활용 / 팁

1. **외출이 잦은 개발자**: Dispatch 설정해두면 이동 중에도 PC 작업 지시 가능
2. **팀 협업**: Channels로 디스코드 채널에 Claude Code를 연결하면, 팀원 누구나 작업 지시 가능
3. **자동화**: 클라우드 스케줄링으로 PR 리뷰, 코드 품질 체크 등을 무인 자동화
4. **1인 창업**: 게리 탄의 스킬 셋업을 적용하면 기획부터 출시까지 체계적으로 진행 가능

## 마무리

Claude Code가 "내 PC에서 돌리는 AI 코딩 도구"에서 **언제 어디서든 접근 가능한 AI 개발 파트너**로 진화하고 있습니다. Open Claw의 성공에서 영감을 받은 기능들(Channels, 원격 제어)이 빠르게 도입되는 걸 보면, AI 코딩 도구들의 기능 경쟁이 본격화되고 있음을 느낄 수 있습니다. 특히 게리 탄의 스킬 셋업은 "AI 도구는 스킬을 어떻게 설정하느냐에 따라 결과가 완전히 달라진다"는 것을 보여주는 좋은 사례입니다.
