# Claude의 새로운 통합 — 타사 모델 게이트웨이 + OpenClaw 재허용 + Project Deal

> 한 줄 요약: Claude Desktop에서 OpenAI 등 다른 모델을 직접 호출할 수 있는 게이트웨이 기능이 추가됐고, 차단됐던 OpenClaw 연동이 다시 허용됐으며, AI끼리 협상하는 Project Deal 실험이 공개됐습니다.

## 핵심 요약
- **타사 추론 게이트웨이**: Claude Desktop과 Claude Code에서 OpenAI 등 다른 모델을 게이트웨이로 호출 가능
- **OpenClaw 재허용**: API 키뿐 아니라 Anthropic 구독으로도 다시 OpenClaw를 사용할 수 있게 됐습니다
- **Project Deal — AI 협상 마켓플레이스**: Claude가 동료 대신 흥정·구매·판매를 수행하는 내부 실험에서 186건, 4,000달러 거래 성사
- **모델 품질이 협상 결과 좌우**: Opus 사용자가 Haiku 사용자보다 더 비싸게 팔고 더 싸게 사는 결과 도출

## 본문

### 타사 추론 게이트웨이 — Claude에서 다른 모델도

Claude Desktop과 Claude Code의 **Connections(커넥션)** 메뉴에 게이트웨이 기능이 추가됐습니다. 이를 통해 Claude 환경에서 OpenAI나 OpenRouter 같은 외부 모델도 호출할 수 있게 됐습니다.

#### 작동 방식

| 기존 | 변경 후 |
|------|--------|
| Claude 모델만 사용 가능 | Claude + 외부 모델 모두 사용 가능 |
| 별도 클라이언트 필요 | Claude 환경에서 통합 |
| - | OpenRouter 등 게이트웨이와 호환 |

#### 추측되는 의도

- **Anthropic 컴퓨팅 부담 분산**: 사용자가 다른 모델을 쓰면 Anthropic 자원 부담이 줄어듭니다
- **생태계 락인 강화**: Claude의 강력한 하네스(Claude Code 등) 환경 안에 사용자를 묶어두는 전략
- **대안 제공**: Claude 모델만 강요하지 않으니 사용자 만족도 향상

> Anthropic의 컴퓨팅 파워 부족 의혹과 맞물려 해석되기도 합니다.

---

### OpenClaw 다시 사용 가능

#### 배경

**OpenClaw**는 Claude를 CLI 환경에서 사용할 수 있게 해주는 서드파티 도구입니다. 이전에 Anthropic이 구독 모델로 OpenClaw 연결을 차단해 큰 논란이 있었습니다.

#### 변경 사항

| 시기 | 정책 |
|------|------|
| 이전 | API 키 연동만 가능, 구독 연동 불가 |
| 현재 | **API + 구독 둘 다 가능** |

#### 의도

- **점유율 확대**: 더 많은 채널에서 Claude를 사용할 수 있게 함
- **API 사용량 증가 유도**: 결과적으로 API 매출 증가 가능
- **개발자 생태계 회복**: 차단으로 잃었던 신뢰 회복

### Project Deal — Claude가 협상하는 AI 마켓플레이스

Anthropic 샌프란시스코 사무실에서 진행된 **내부 실험 프로젝트**입니다.

#### 어떤 실험인가

| 항목 | 내용 |
|------|------|
| 형태 | 사내 직원용 마켓플레이스 (당근마켓 같은 것) |
| 특이점 | 사람 대신 **Claude가 흥정·구매·판매 대행** |
| 사용자 행동 | 물건 등록만 하면 끝 |
| AI 행동 | 가격 흥정, 거래 합의, 매칭 자동 처리 |

#### 실험 결과

- **186건의 거래 성사**
- **총 거래액 4,000달러 초과**
- 설문 조사: 참가자 **거의 절반이 향후 이런 서비스에 지불 의향 있음**
- 평가: "Claude의 거래가 공정해 보인다"

#### 흥미로운 발견 — 모델 품질이 결과에 영향

| 모델 사용자 | 자전거 판매 결과 |
|-----------|---------------|
| Opus (강력 모델) | **더 비싸게 판매** |
| Haiku (작은 모델) | 더 싸게 판매 |

> 비싼 에이전트한테 맡기면 더 좋은 거래를 만들어낼 수 있다는 흥미로운 결과가 나왔습니다.

#### 더 흥미로운 점

설문 조사에 참여한 사람들은 **이 차이를 알아채지 못했습니다**. 모두가 "AI가 잘해줬네"라고 생각했습니다.

### 시사점

#### AI 중계자(Intermediary)의 가능성

협상이라는 인간 고유의 영역을 AI가 대신할 수 있는 가능성이 입증됐습니다.

- 인간 중개자 → AI 중개자
- 토큰 비용만으로 협상·매칭 서비스 가능
- 공정성도 인정받음

#### 모델 격차의 사회적 영향

좋은 모델을 가진 사람과 그렇지 않은 사람 간에 협상 결과 격차가 발생할 수 있습니다. 이는 AI 시대의 새로운 격차 이슈로 떠오를 수 있습니다.

## 실전 활용 / 팁

- **Claude 통합 워크플로우**: 게이트웨이 기능으로 Claude Code 안에서 GPT 5.5나 Gemini 같은 다른 모델도 함께 사용해보세요. 작업별로 최적 모델을 선택할 수 있습니다
- **OpenClaw 활용**: Claude 구독자라면 OpenClaw로 CLI 환경에서 Claude를 더 효율적으로 사용할 수 있습니다
- **AI 협상 자동화 가능성**: Project Deal의 결과는 향후 중고 거래, B2B 매칭, 부동산 등 다양한 분야에서 AI 중개 서비스가 등장할 가능성을 시사합니다

## 마무리

Claude의 이번 업데이트들은 **개방성과 통합성** 방향으로 움직이고 있다는 신호입니다. 다른 모델을 자기 환경 안에서 사용할 수 있게 하고, 차단했던 서드파티를 다시 받아들였습니다. Project Deal은 AI가 단순히 정보 제공이나 코드 작성을 넘어 **인간 사회의 거래·협상 영역까지 대체할 수 있다**는 가능성을 보여준 흥미로운 실험입니다. 좋은 모델이 좋은 결과를 만든다는 점은 곧 "AI 격차"라는 새로운 사회적 문제로 이어질 수 있다는 점도 함께 생각해볼 만합니다.
